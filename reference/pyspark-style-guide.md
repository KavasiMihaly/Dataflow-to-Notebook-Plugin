# PySpark Style Guide for Fabric Notebooks

Standards for PySpark code generated by Fabric agents. Ensures consistency across bronze, silver, and gold layer notebooks.

---

## Import Conventions

```python
# Always alias pyspark.sql.functions as F
from pyspark.sql import functions as F
from pyspark.sql.types import *
from pyspark.sql.window import Window
from delta.tables import DeltaTable
```

**Rules:**
- Always use `F.` prefix for functions: `F.col()`, `F.lit()`, `F.current_timestamp()`
- Never use `from pyspark.sql.functions import *` (pollutes namespace, causes ambiguity with Python built-ins like `sum`)
- Import `DeltaTable` by name, not `from delta.tables import *`

---

## Column Naming

All columns use **snake_case** throughout all layers.

| Category | Pattern | Examples |
|----------|---------|----------|
| Primary keys | `<entity>_id` | `customer_id`, `order_id` |
| Foreign keys | `<related_entity>_id` | `customer_id`, `product_id` |
| Dates | `<event>_date` | `order_date`, `ship_date` |
| Timestamps | `<event>_at` | `created_at`, `updated_at` |
| Booleans | `is_<condition>` | `is_active`, `is_deleted` |
| Counts | `<noun>_count` | `order_count`, `line_count` |
| Amounts | `<noun>_amount` | `total_amount`, `tax_amount` |
| Metadata | `_<purpose>` (underscore prefix) | `_load_timestamp`, `_source_file` |

---

## Table Naming by Layer

| Layer | Pattern | Examples |
|-------|---------|----------|
| Bronze | `bronze_<source>` | `bronze_customers`, `bronze_orders` |
| Silver | `silver_<entity>` | `silver_customers`, `silver_orders` |
| Gold (dimensions) | `dim_<entity>` | `dim_customer`, `dim_product` |
| Gold (facts) | `fct_<process>` | `fct_sales`, `fct_inventory` |
| Gold (aggregates) | `agg_<description>` | `agg_sales_by_region` |

## DataFrame Variable Naming

Use `df_` prefix with descriptive purpose:

```python
df_raw = spark.read.format("csv").load(...)       # Raw source data
df_clean = df_raw.filter(F.col("id").isNotNull())  # After cleaning
df_dedup = df_clean.dropDuplicates(["order_id"])   # After deduplication
df_enriched = df_dedup.join(df_lookup, ...)        # After enrichment
df_final = df_enriched.select(...)                 # Final output
```

## Method Chaining Style

Use backslash continuation with one operation per line, indented by 4 spaces:

```python
# Good - one operation per line, aligned
df_result = df_raw \
    .filter(F.col("customer_id").isNotNull()) \
    .withColumn("order_date", F.to_date(F.col("order_date_str"), "yyyy-MM-dd")) \
    .withColumn("amount", F.col("amount").cast("decimal(18,2)")) \
    .select("customer_id", "order_date", "amount")

# Good - write operations
df_result.write.format("delta") \
    .mode("append") \
    .option("mergeSchema", "true") \
    .saveAsTable("bronze_orders")
```

```python
# Bad - everything on one line
df_result = df_raw.filter(F.col("customer_id").isNotNull()).withColumn("order_date", F.to_date(F.col("order_date_str"), "yyyy-MM-dd")).select("customer_id", "order_date")
```

## Notebook Cell Comments

Each cell should start with a markdown header or a descriptive Python comment:

```python
# --- Read Source Data ---
df_raw = spark.read.format("csv") \
    .option("header", "true") \
    .load("Files/raw/customers.csv")
```

Use `# ---` delimiters for major sections within a cell.

## Error Handling Patterns

### Validation Assertions (preferred for pipeline notebooks)
```python
# Validate row count
row_count = df_result.count()
assert row_count > 0, f"FAIL: No rows written to bronze_customers"
print(f"PASS: {row_count} rows written")
```

### Try/Except (for external source operations)
```python
try:
    df_raw = spark.read.format("csv") \
        .option("header", "true") \
        .load(source_path)
except Exception as e:
    print(f"ERROR: Failed to read source: {e}")
    notebookutils.notebook.exit(f"FAILED: {e}")
```

## String Literals

- Use **double quotes** for SQL strings and table names: `"bronze_customers"`
- Use **f-strings** for dynamic values: `f"bronze_{source_name}"`
- Use **triple quotes** for multi-line SQL in `spark.sql()`:

```python
df = spark.sql(f"""
    SELECT *
    FROM bronze_{source_name}
    WHERE _load_timestamp > '{last_watermark}'
""")
```

## Constants and Configuration

Define at the top of the notebook in a Parameters cell:

```python
# Parameters
source_name = "customers"
load_mode = "append"
bronze_lakehouse = "lh_bronze"
```

---

## Avoid

- `from pyspark.sql.functions import *` - always use `F.` alias
- `collect()` on large DataFrames - use `first()`, `take()`, or `limit()` instead
- `.count()` in loops - cache the DataFrame or use accumulator
- Hard-coded credentials - use `notebookutils.credentials.getSecret()`
- `monotonically_increasing_id()` for surrogate keys in production - use `row_number()` or hash-based keys for deterministic results
